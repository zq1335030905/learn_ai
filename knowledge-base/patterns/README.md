## Pattern 条目结构

每个 Pattern 应包含：
1. **问题场景**：什么情况下需要这个模式
2. **核心结构**：关键组件和数据流
3. **最小实现**：可运行的代码示例
4. **关键取舍**：性能/复杂度/兼容性权衡
5. **失败模式**：常见坑点和规避方法
